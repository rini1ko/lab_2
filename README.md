# Лабораторна робота: Скінченні автомати та регулярні вирази

**Мета:** Імплементація рушія регулярних виразів на основі скінченних автоматів (FSA).

## Код до лабораторної роботи
Вихідний код знаходиться у файлах репозиторію:
- `regex.py` — основна логіка скінченного автомата та класів станів.
- `demo.py` — скрипт для демонстрації роботи та інтерактивного тестування.

## Коротке пояснення імплементації
Програму побудовано з використанням об'єктно-орієнтованого патерну **State**. Кожен елемент регулярного виразу представлений як окремий стан скінченного автомата.

### Архітектура станів
- `AsciiState` — розпізнавання латини (верхнього та нижнього регістрів) і цифр.
- `DotState` — обробка оператора `.`, що приймає будь-який один символ.
- `StarState` — обгортка для оператора `*`, яка дозволяє попередньому стану повторюватися нуль або більше разів.
- `PlusState` — обгортка для оператора `+`, яка вимагає від попереднього стану щонайменше одного входження.
- `StartState` та `TerminationState` — технічні стани, що позначають початкову та кінцеву точки перевірки відповідно.

### Алгоритм обробки
Метод `check_string` реалізує алгоритм DFS. Оскільки наявність операторів `*` та `+` перетворює автомат на недетермінований, алгоритм DFS забезпечує бектрекінг. Якщо поточний шлях перевірки не веде до кінцевого стану (наприклад, стан поглинув забагато символів або необхідний символ відсутній), алгоритм повертається до попереднього вузла розгалуження та перевіряє інші гілки виконання.

Для уникнення конфліктів між екземплярами класів, масиви `next_states` ініціалізуються індивідуально в конструкторі `__init__` для кожного стану, що гарантує структурну цілісність графа.

## Інструкції до запуску
Для роботи програми необхідний встановлений Python3. Додаткові бібліотеки не потрібні.

1. Завантажте файли `regex.py` та `demo.py` у спільну директорію.
2. Відкрийте термінал у цій директорії.
3. Виконайте команду для запуску демонстрації:
   ```bash
   python test_regex.py
